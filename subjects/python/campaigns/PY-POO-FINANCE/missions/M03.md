---
## Objective
Separar la lógica de guardado (CSV) de la lógica de negocio, creando una clase `TransactionRepository`.

---
## Description
Actualmente `storage/transactions_csv.py` tiene funciones sueltas. Vamos a convertirlo en una clase que gestione la entrada/salida.
Esto permitirá en el futuro cambiar CSV por SQL sin romper el resto del programa.

---
## Steps
1. Crear `storage/repository.py`.
2. Definir la clase `TransactionRepository`.
3. Mover `save_transactions` y `append_transaction` como métodos de esa clase.
4. (Opcional) Hacer que el `filename` sea un atributo de la clase, no un parámetro de cada función.
5. Actualizar `main.py` para usar una instancia del repositorio.

---
## Completion Criteria
- [ ] Existe una clase `TransactionRepository`.
- [ ] `main.py` instancia el repo y lo usa para guardar.
