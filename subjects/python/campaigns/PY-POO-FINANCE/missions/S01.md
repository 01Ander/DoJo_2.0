## Objective
Asegurar que la lógica financiera no tenga errores creando tests automáticos.

---
## Description
El dinero no perdona errores. Escribirás tests para verificar que no se pueda retirar más del saldo o pagar deuda negativa.

---
## Steps
1. Instalar `pytest`.
2. Crear carpeta `tests/`.
3. Crear `tests/test_accounts.py`.
4. Testear:
    - Depósito aumenta saldo.
    - Retiro sin fondos lanza error.
    - Tarjeta calcula bien el `available`.

## Completion Criteria
- [ ] Ejecutar `pytest` muestra al menos 5 tests pasando en verde.
